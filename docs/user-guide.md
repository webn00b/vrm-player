## Для чего нужен проект

`VRM Player` позволяет:

- загрузить VRM-аватара;
- проигрывать на нём `.bvh` анимации;
- накладывать мокап с камеры или видео;
- записывать результат в `.bvh`;
- экспортировать текущую позу как один кадр BVH.

## Быстрый старт

### Требования

- Node.js 18+
- npm

### Установка

```bash
npm install
npm run dev
```

Приложение открывается на [http://127.0.0.1:5333](http://127.0.0.1:5333).

## Структура файлов

### VRM-модель

- папка `models/`
- используется первый `.vrm` по алфавиту

Пример:

```text
models/
  avatar.vrm
```

### BVH-анимации

- папка `animations/`
- используются все `.bvh`, найденные при старте

Пример:

```text
animations/
  01-idle.bvh
  02-walk.bvh
  03-wave.bvh
```

## Основные сценарии

### 1. Проигрывание BVH

1. Положите `.vrm` в `models/`.
2. Положите `.bvh` в `animations/`.
3. Запустите проект.
4. Во вкладке `Main` включите `Show model`.
5. Перетащите анимации из `Library` в `Queue`.

Что можно проверить в этом режиме:

- корректность retarget-а;
- кроссфейды между клипами;
- поведение модели на разных BVH.

### 2. Live мокап с камеры

Во вкладке `Video`:

1. Нажмите `Start`.
2. Разрешите доступ к камере.
3. Включите `Show model`, если нужно видеть аватара.
4. Нажмите `Rec`, если хотите записать результат в `.bvh`.

Полезные переключатели:

- `Mirror mode` — селфи-режим.
- `Face tracking` — анимация лица.
- `Hip position` — перенос таза по позиции.
- `1€ smoothing` — сглаживание landmarks.
- `Wrist + fingers priority` — кисти и пальцы остаются самым верхним слоем.

### 3. Мокап из видеофайла

Во вкладке `Video`:

1. Нажмите `Load`.
2. Выберите видеофайл.
3. Дождитесь обработки.
4. Проект автоматически скачает `.bvh` и добавит его в текущую очередь.

Этот режим удобен, когда нужно сравнить:

- исходное видео;
- live/debug skeleton;
- записанный BVH;
- повторный playback того же BVH на модели.

### 4. Пошаговый разбор видео

Если открыт видеофайл, можно:

- ставить видео на паузу;
- шагать вперёд и назад по кадрам;
- собирать ручной буфер кадров;
- выгружать буфер как BVH.

Кнопки:

- `⏸` / `▶` — пауза и продолжение.
- `⏮` — шаг назад.
- `⏭` — шаг вперёд.
- `💾` — добавить текущий кадр в ручной буфер.
- `⬇` — скачать ручной буфер как `.bvh`.

### 5. Экспорт одной позы

Во вкладке `Video` есть кнопка:

- `Current pose -> Export .bvh`

Она выгружает текущее состояние аватара как отдельный `1-frame BVH`.

Это полезно, когда нужно:

- сохранить проблемный кадр;
- передать конкретную позу в другой инструмент;
- сравнить одно положение до и после правки solver-а.

## Что важно понимать про итоговую позу

Каждый кадр строится слоями:

1. BVH-анимация
2. idle / procedural позы
3. live mocap
4. manual offsets
5. финальный overlay кистей и пальцев
6. validator
7. VRM update / микродвижения

Практический смысл:

- если live mocap не виден, его мог перетереть более поздний слой;
- если ошибка уже заметна на debug skeleton, проблема раньше, чем ретаргет на модель;
- если debug skeleton выглядит нормально, а аватар нет, значит искать нужно в solver-е, rest correction или validator-е.

## Ограничения

Проект хорошо подходит для отладки, но важно помнить:

- MediaPipe может шуметь по глубине;
- пальцы и кисти не будут идеально совпадать с видео на каждом кадре;
- сильная разница пропорций между перформером и аватаром влияет на retarget;
- upper-body видео плохо подходят для честного leg IK;
- нестандартный rest pose модели иногда требует дополнительной компенсации.
